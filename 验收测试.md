# 分布式文件管理系统 - 验收测试指南（含RBAC）

## 测试环境准备

### 1. 基础服务启动
```bash
# 1. 启动Docker容器（MySQL、Redis、Zookeeper）
cd e:\trea-workspace\FileManager\docker
docker-compose up -d

# 2. 编译项目（首次或代码有改动时）
cd e:\trea-workspace\FileManager
mvn install -DskipTests -q

# 3. 启动后端服务（新开终端窗口）
cd e:\trea-workspace\FileManager
mvn spring-boot:run -pl file-manager-web

# 4. 启动前端服务（新开终端窗口）
cd e:\trea-workspace\FileManager\file-manager-frontend
npm run dev
```

### 2. 访问地址
- **前端界面**：http://localhost:5173
- **后端API**：http://localhost:8080
- **默认账号**：admin / admin123
- **API文档**：http://localhost:8080/api/doc.html

### 3. 预置数据说明

| 账号 | 密码 | 角色 | 权限范围 |
|------|------|------|----------|
| admin | admin123 | ADMIN（管理员） | 全部权限 |
| （注册新用户自动获得） | （自设） | USER（普通用户） | 仅文件操作权限 |

**测试B - 普通用户登录**

| 1 | 先注册一个新用户（如 testuser / 123456） | 注册成功 |

| 3 | 尝试直接访问 /users | 页面无内容或提示无权限 |

### 2.2 API接口权限校验

使用Postman或浏览器开发者工具测试：

| 测试场景 | 方法 | URL | 用admin Token | 用USER Token | 预期 |
|----------|------|-----|---------------|--------------|------|
| 文件列表 | GET | /file/list | ✅ 200 | ✅ 200 | 都能访问 |
| 用户列表 | GET | /auth/users | ✅ 200 | ❌ 403 | 仅ADMIN |
| 角色列表 | GET | /role/list | ✅ 200 | ❌ 403 | 仅ADMIN |
| 节点列表 | GET | /node/online | ✅ 200 | ❌ 403 | 仅ADMIN |
| 文件上传 | POST | /file/upload | ✅ 200 | ✅ 200 | 都能上传 |
| 删除文件 | DELETE | /file/{key} | ✅ 200 | ✅ 200 | 都能删除自己的 |

### 2.3 权限码验证（数据库）
```sql
-- 连接MySQL查看初始化的权限数据
docker exec -it fm-mysql mysql -u root -proot_password file_manager

-- 查看所有权限
SELECT id, permission_code, permission_name FROM sys_permission;
-- 预期输出16条记录：
-- file:upload, file:download, file:delete, file:list
-- user:view, user:create, user:edit, user:delete, user:assign-role, user:reset-password
-- role:view, role:create, role:edit, role:delete, role:assign-permission
-- node:view

-- 查看ADMIN角色拥有所有权限
SELECT COUNT(*) FROM sys_role_permission WHERE role_id = 1;
-- 预期：16

-- 查看USER角色仅拥有文件权限
SELECT p.permission_code FROM sys_permission p
INNER JOIN sys_role_permission rp ON p.id = rp.permission_id
WHERE rp.role_id = 2;
-- 预期：仅 file:* 的4条
```

## 七、完整业务流程测试

### 流程1：管理员创建运营人员账号
```
1. admin 登录
2. 进入 用户管理 → 新建用户 → yunying1 / 123456 / 运营小王
3. 进入 角色管理 → 新建角色 → OPERATOR / 运营人员 / 文件+用户查看权限

3. 用普通用户(testuser)重新登录

```

## 八、数据库完整性检查

```sql
-- 连接数据库
docker exec -it fm-mysql mysql -u root -proot_password file_manager

-- 1. 检查所有表存在且注释中文正常
SHOW TABLES;
-- 预期10张表：sys_user, sys_role, sys_permission, sys_user_role,
--       sys_role_permission, file_metadata, file_chunk, storage_node,
--       operation_log（如有）

-- 2. 检查权限数据量
SELECT COUNT(*) as total_perms FROM sys_permission;   -- 预期: 16
SELECT COUNT(*) as total_roles FROM sys_role;          -- 预期: 2+
SELECT COUNT(*) as total_users FROM sys_user WHERE deleted='0'; -- 预期: 1+

-- 3. 检查admin权限完整性
SELECT COUNT(*) as admin_perms FROM sys_role_permission WHERE role_id = 1;
-- 预期: 16 (全部权限)

-- 4. 检查USER角色权限
SELECT p.permission_code FROM sys_permission p
INNER JOIN sys_role_permission rp ON p.id = rp.permission_id
WHERE rp.role_id = 2 ORDER BY p.permission_code;
-- 预期仅4条: file:delete, file:download, file:list, file:upload

-- 5. 检查中文注释无乱码
SELECT TABLE_NAME, TABLE_COMMENT FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'file_manager' AND TABLE_NAME LIKE 'sys_%';
-- 预期全部中文且可读
```
