## 整体目标
开发一个分布式文件管理系统，支持文件的上传、下载、删除、查询等操作。
文件管理系统需要考虑以下方面：
- 分布式存储：将文件存储在多个节点上，每个节点负责存储部分文件。
- 高可用：通过复制文件到多个节点上，实现文件的高可用。
- 高并发：能够处理多个用户同时操作文件，确保系统的高并发。
- 安全性：对文件进行加密存储，防止文件被未授权访问。
- 可扩展性：能够方便地扩展系统，支持新增节点。



## 技术选型
- 前端：Vue.js + Element UI
- 后端：Spring Boot + MyBatis-Plus
- 数据库：MySQL
- 缓存：Redis
- 分布式协调：Zookeeper
- 分布式锁：Redission
- 加密：AES加密
- 认证：JWT
- 授权：RBAC
- 日志：Logback

## 本地开发环境
- docker：用于快速搭建开发环境，包括数据库、缓存、分布式协调等。
- maven：用于构建项目依赖。
- java 21
- git：用于版本控制。
