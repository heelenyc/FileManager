### 二、核心模块设计

#### 1. 分布式存储

| 项目 | 方案 |
|------|------|
| 文件分片 | 大文件按固定大小(如 64MB)分片，小文件直接存储 |
| 节点选择 | 基于 Zookeeper 维护节点列表，一致性哈希决定文件块存储位置 |
| 元数据管理 | MySQL 存储文件元信息(文件名、大小、分片位置、副本节点等) |
| 节点注册 | 存储节点启动时向 Zookeeper 注册临时节点，心跳保活 |

**核心数据模型：**

```sql
-- 文件元数据
file_metadata(id, file_name, file_size, content_type, aes_key_encrypted,
              upload_user, created_at, updated_at, deleted)

-- 文件分片
file_chunk(id, file_id, chunk_index, chunk_size, chunk_hash,
           storage_node, replica_nodes)

-- 存储节点
storage_node(id, node_host, node_port, status, available_space,
             registered_at, last_heartbeat)
```

#### 2. 高可用

- **多副本策略**：每个文件块默认存储 3 个副本，分布在不同节点
- **副本放置**：优先选择不同机架/可用区的节点
- **故障检测**：Zookeeper 临时节点监听，节点下线自动触发副本迁移
- **数据修复**：定时任务检测副本数不足的文件块，自动补充副本

#### 3. 高并发

- **上传**：分片并发上传 + 断点续传，Redis 记录上传进度
- **下载**：多副本并行读取，选择最近/负载最低的节点
- **分布式锁**：Redission 实现文件操作互斥，防止并发写冲突
- **缓存**：Redis 缓存热点文件元数据和下载链接，减轻 DB 压力
- **限流**：网关层基于令牌桶限流

#### 4. 安全性

| 层面 | 方案 |
|------|------|
| 传输安全 | HTTPS + 文件上传/下载接口签名 |
| 存储加密 | AES-256 加密文件内容，每个文件独立密钥，密钥用用户密钥加密后存 DB |
| 认证 | JWT Token，Access Token(短期) + Refresh Token(长期) |
| 授权 | RBAC 模型：用户 → 角色 → 权限，文件级别权限控制 |
| 审计 | Logback 记录所有文件操作日志，包含操作人、时间、IP |

#### 5. 可扩展性

- **水平扩容**：新存储节点注册到 Zookeeper，一致性哈希自动迁移部分数据
- **服务无状态**：业务服务层无状态设计，可随时增减实例
- **虚拟节点**：一致性哈希使用虚拟节点，保证数据分布均匀

### 三、核心流程

#### 文件上传流程
```
1. 客户端请求上传 → JWT 鉴权
2. 文件服务生成文件唯一ID + AES密钥
3. 大文件分片 → 一致性哈希选择存储节点(主+2副本)
4. 并发上传分片到各节点，AES加密后落盘
5. 写入 file_metadata + file_chunk 到 MySQL
6. Redis 缓存文件元数据
7. 返回上传结果
```

#### 文件下载流程
```
1. 客户端请求下载 → JWT 鉴权 + RBAC 权限校验
2. 查 Redis 缓存 → 未命中则查 MySQL
3. 根据分片信息，选择负载最低的副本节点
4. 从存储节点读取加密分片 → AES解密 → 合并
5. 流式返回给客户端
```

### 四、项目结构

```

├── docker/                    # Docker 编排
│   ├── docker-compose.yml     # MySQL + Redis + Zookeeper
```

### 五、Docker 开发环境

```yaml
# docker-compose.yml 核心配置
services:
  mysql:
    image: mysql:8.0
    ports: ["3306:3306"]
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: file_manager

  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]

  zookeeper:
    image: zookeeper:3.9
    ports: ["2181:2181"]
```

### 六、关键技术点

| 问题 | 解决方案 |
|------|---------|
| 一致性哈希数据倾斜 | 引入虚拟节点(每个物理节点 150 个虚拟节点) |
| 并发上传同一文件 | Redission 分布式锁，锁 key = `file:upload:{fileId}` |
| 大文件传输性能 | 分片并发上传 + 断点续传 + 流式下载 |
| 节点故障数据丢失 | 3 副本 + Zookeeper 心跳检测 + 自动迁移 |
| 密钥管理 | 每文件独立 AES 密钥，用主密钥加密后存 DB，主密钥存环境变量 |
| 热点文件压力 | Redis 缓存元数据 + 多副本负载均衡读取 |
